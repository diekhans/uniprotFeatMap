UniProt Transcript Annotations

This track decorator set consist of UniProt annotations on individual GENCODE V45 protein-coding transcripts.

Annotations from both the SwissProt and TrEMBL subsets of UniProt are available. The UniProt canonical transcripts have annotations added to the transcript based on their protein coordinates. Non-UniProt canonical transcripts have the annotation coordinates projected via the genome coordinates.

Disputed projected annotations are indicated in the display, as described below. This provides a visualization that a transcript may produce a protein that may have altered function or is non-functional.

Display Conventions and Configuration

There are two types of annotations on transcripts, implemented as track decorators:

  • Feature annotations: indicate the location of the UniProt features
  • Disruption annotations: indicate the location of disruption in features projections

Feature annotations

Feature annotations have their base items colored by type:

  • signal peptide domains
  • extracellular domains
  • transmembrane domains
  • cytoplasmic domains
  • topological domain
  • zinc finger region
  • disulfide bond
  • lipoification modification sites
  • glycosylation modification sites
  • phosphorylation sites
  • modified residues
  • other SwissProt
  • other TrEMBL

Feature annotations are outlined based on whether they are UniProt canonical or non-canonical, or if the feature is disrupted:

  • canonical
  • noncanonical
  • disrupted

Disruption annotations

Disruption annotations indicate where a feature is broken either by an insertions or deletions create by alternative splicing. In addition to outlining the disrupted features in red, additional annotations are added to show the location in the feature where the disruption occurs. These are represented by:

  • A red block annotation indicated the range where there is an insertion
  • A red triangle pointing to the points were a deletion occurred. Internal deletions will have two triangles pointing to each end of the deletion and truncations will have a single triangle.

Mouseover metadata

  • UniProt data set: either SwissProt (manual annotation) or TrEMBL (automated annotation)
  • UniProt acc: UniProt accession with link to UniProt entry.
  • feature type: UniProt feature type
  • feature category: a more general category for the feature assigned by UCSC for filtering and display purposes
  • transcript category:
    • canonical: transcript accession and version is listed by UniProt as match the protein
    • canonical_diff_version: transcript accession is listed by UniProt as matching the protein however the version is different
    • non-canonical : transcript that is not listed by UniProt that has the features projected on it
  • feature status:
    • for feature annotations: complete or disrupted
    • for disruption annotations: insert, del_5p for 5' deletions, del_3p for 3' deletions, or del_int for internal deletions
  • description: for feature annotations, description from UniProt; for disruption annotations, describes the type and size of the disruption.
  • comment: comment from UniProt
  • disease: disease name for disease-related variants

Display control

These controls allow the user to adjust the placement of the annotations. Block annotations (features and insertion disruptions) and glyph annotations (deletion disruptions) are controlled separately.

  • hide: annotations are not shown
  • overlay: annotations are placed over the transcripts
  • adjacent: annotations are placed below the transcripts.

Filters

These filters are available to control what annotations are displayed. An annotation must match all filters to be displayed.

  • Annotation category: general category of annotation (default Domain)
  • Uniprot dataset: SwissProt or TrEMBL dataset (default SwissProt)
  • Annotation type: feature or disruption (default All)

Methods

  1. Convert the GENCODE transcript annotations on the genome to the implied alignments for use in projected (transGenomeAln)
  2. Produce alignments of UniProt proteins to GENCODE mRNA transcripts (biotype XXX of protein_coding), resulting protein to mRNA alignments (protTranAln) with a two-step process:
    1. UniProt proteins are aligned to their corresponding canonical mRNA transcript in GENCODE using translated BLAT. The UTR regions of the mRNA are hard-masked to prevent protein alignment to similar sequences outside of the CDS. The canonical mRNA is matched, but the version number is ignored.
    2. Produce protein-to-mRNA transcript alignments for the remaining protein-coding GENCODE transcripts that are not UniProt canonical transcripts by projecting the canonical protein to mRNA alignment through the genome. Projections to UTR regions are clipped.
  3. Project the UniProt feature annotations on the proteins to the associated mRNAs, and then project those to the genomic coordinates for each mRNA (annotGenomeAln). The projection pipeline is thus: 
          annotation -> protTranAln ->  transGenomeAln => annotGenomeAln
  4. Create a Track Decorator BED file associating the feature annotations map to each transcript with the transcript. Associated metadata for each feature and identify disruptions in features. Block decorations on the transcripts are created for each feature, and triangular glyph decorations are used to indicate the points of feature disruption.
